## 一、CS:APP 第三章 3.6 ~ 3.11 学习总结

### 1.1 控制流与分支（3.6）

#### 条件码

CPU 通过四个条件码标志位来判断运算结果：

- **CF（Carry Flag）**：进位标志，用于无符号溢出
- **ZF（Zero Flag）**：零标志，结果为零时置 1
- **SF（Sign Flag）**：符号标志，结果为负时置 1
- **OF（Overflow Flag）**：溢出标志，用于有符号溢出

`cmp` 和 `test` 指令专门用来设置条件码而不保存运算结果。`cmp` 等价于减法，`test` 等价于按位与。

#### 跳转与分支

C 语言中的 `if-else`、三元运算符、循环最终都会翻译成 `jX` 条件跳转指令。编译器通常会把条件判断"反过来"，例如 `if (a > b)` 常常对应汇编中的 `jle`（小于等于则跳转到 else）。

`setX` 指令把比较结果变成 0 或 1，这正是 C 语言关系表达式（如 `int x = (a > b);`）的底层实现。

#### 循环结构

`do-while`、`while`、`for` 在汇编层面殊途同归。`do-while` 最自然；`while` 和 `for` 常被编译器翻译成 "jump-to-middle" 或 guarded-do 形式。guarded-do 在循环开始前先做条件判断，对 CPU 分支预测更友好。

#### Switch 与跳转表

当 `case` 分支较多且取值连续时，编译器会生成**跳转表**（Jump Table），把 O(n) 的条件判断优化为 O(1) 的数组查表操作。间接跳转指令 `jmp *.L4(,%rdi,8)` 是跳转表的典型特征。

#### CMOV 条件传送

现代 CPU 使用流水线技术，分支预测失败的代价很高。`cmov` 指令通过避免分支来减少预测失败的惩罚，但当分支有副作用或两个分支计算量差异大时不能使用。

### 1.2 过程调用与运行时栈（3.7）

#### 栈的基本模型

x86-64 的栈向**低地址方向增长**。`%rsp` 指向栈顶，`push` 减小 `%rsp`，`pop` 增大 `%rsp`。

`call` 指令把返回地址压栈后跳转；`ret` 从栈顶弹出返回地址并跳转回去。

#### 参数传递

System V AMD64 ABI 规定：前 6 个整型/指针参数通过寄存器 `%rdi`、`%rsi`、`%rdx`、`%rcx`、`%r8`、`%r9` 传递，返回值放在 `%rax`。

口诀：**Diane's Silk Dress Cost 89**（rdi, rsi, rdx, rcx, r8, r9）。

#### 寄存器保存约定

- **调用者保存**：`%rax`、`%rcx`、`%rdx`、`%rsi`、`%rdi`、`%r8~%r11`
- **被调用者保存**：`%rbx`、`%rbp`、`%r12~%r15`

被调用者保存的寄存器如果使用，必须在返回前恢复原值。

#### 超过 6 个参数的传递

第 7 个及以后的参数通过栈传递，按从右到左的顺序压栈。`call` 之后返回地址占 8 字节，所以第 7 个参数位于 `16(%rsp)` 处。

#### 结构体返回值

- 小于等于 16 字节的结构体：通过 `%rax`（和 `%rdx`）返回
- 大于 16 字节的结构体：调用者先分配内存，把地址作为隐藏的第 0 个参数传给被调函数

### 1.3 数组分配与访问（3.8）

数组在内存中连续存放。`A[i]` 等价于 `*(A + i)`，指针运算会自动进行类型缩放。

多维数组按**行优先**存储。`A[i][j]` 的地址计算公式为：

```
&A[i][j] = A + (i * C + j) * sizeof(T)
```

定长数组在编译时即可确定地址，VLA（变长数组）需要在运行时动态分配栈空间。

#### VLA 是什么

VLA（Variable Length Array）是大小在运行时确定的局部数组，仍然存放在栈上。例如：

```c
void foo(int n) {
    int arr[n];   // VLA
}
```

VLA 的缺点是可移植性差（MSVC 不支持）、栈空间有限、不能返回。大数组时更建议使用 `malloc`。

---

### 1.4 异质数据结构（3.9）

#### 结构体与内存对齐

结构体大小不等于成员大小之和，编译器会插入填充字节以满足对齐要求。四大原则：

1. 成员按自身大小对齐
2. 结构体总大小必须是最宽成员的整数倍
3. 结构体起始地址必须是最宽成员的整数倍
4. 嵌套结构体按自己的最大对齐值对齐

#### 联合（Union）

`union` 允许不同成员共享同一块内存，常用于 Type Punning，例如窥探 IEEE 754 浮点数的二进制编码。

---

### 1.5 对齐与填充（3.10）

对齐不仅发生在结构体内部，栈上的局部变量、`malloc` 返回的堆内存、整个程序的内存布局都需要对齐。这是为了满足 CPU 缓存行和硬件总线的要求，未对齐访问可能降低性能甚至引发异常。

GCC/Clang 提供了 `__attribute__((aligned(n)))` 来强制指定对齐方式，标准 C11 中对应的是 `_Alignas(n)`。

---

### 1.6 浮点代码（3.11）

#### IEEE 754 标准

浮点数由三段组成：

- **符号位**：决定正负
- **指数位**：带偏移量表示
- **尾数位**：带隐藏位表示有效数字

通过这个标准可以理解规格化数、非规格化数、无穷大和 NaN 的编码。

#### SSE/AVX

现代 x86-64 使用 `%xmm0~%xmm15` 寄存器组进行浮点运算，`movss` 和 `movsd` 是单精度和双精度浮点传送的核心指令。

## 二、运行时栈的汇编实现

### 2.1 栈帧构建的典型模式

```asm
foo:
    pushq %rbp           # 保存旧帧指针
    movq  %rsp, %rbp     # 设置新帧指针
    subq  $16, %rsp      # 分配局部变量空间

    # 函数体

    movq  %rbp, %rsp     # 恢复栈指针
    popq  %rbp           # 恢复旧帧指针
    ret
```

现代编译器优化后通常会省略 `%rbp`，直接用 `%rsp` 访问局部变量。

### 2.2 栈上数组的实现

栈上的数组就是栈帧中一段连续的内存。普通数组编译时直接分配固定空间：

```asm
subq  $16, %rsp        # 分配 4 个 int
movl  $10, -16(%rbp)   # arr[0] = 10
movl  $20, -12(%rbp)   # arr[1] = 20
```

VLA 需要运行时计算大小：

```asm
movq  %rdi, %rax       # n
salq  $2, %rax         # n * 4
subq  %rax, %rsp       # 分配 n*4 字节
```

### 2.3 16 字节栈对齐

调用 `call` 指令时，`%rsp` 必须是 16 的倍数。编译器会通过调整栈空间来保持对齐。
