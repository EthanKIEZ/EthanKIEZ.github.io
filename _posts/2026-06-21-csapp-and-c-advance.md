### 2.2 条件传送对比分支跳转

**知识点**：`cmov` 条件传送、`jcc` 条件跳转、分支预测

在实现条件判断时，编译器通常有两种选择：**条件跳转**（`jcc`）和**条件传送**（`cmov`）。初学者很容易混淆，下面从三个角度对比。

#### 汇编实现不同

| 方式 | 汇编指令 | 工作方式 |
|------|---------|---------|
| 分支跳转 | `je`、`jne`、`jg`、`jl` 等 | 根据条件跳转到不同代码位置 |
| 条件传送 | `cmove`、`cmovg`、`cmovl` 等 | 根据条件决定是否把源操作数传送到目的寄存器 |

#### C 代码对比

假设要实现：

```c
int max = (a > b) ? a : b;
```

**用分支跳转实现：**

```asm
    cmpq %rsi, %rdi       # 比较 a 和 b
    jle  .L_else          # 如果 a <= b，跳到 else
    movq %rdi, %rax       # max = a
    jmp  .L_done
.L_else:
    movq %rsi, %rax       # max = b
.L_done:
```

**用条件传送实现：**

```asm
    cmpq %rsi, %rdi       # 比较 a 和 b
    movq %rdi, %rax       # 先假设 max = a
    cmovle %rsi, %rax     # 如果 a <= b，把 b 传送给 max
```

#### 应用环境不同

| 场景 | 推荐方式 | 原因 |
|------|---------|------|
| 简单赋值、求最值 | `cmov` | 避免分支预测失败 |
| 分支有副作用 | `jcc` | `cmov` 会同时计算两个分支 |
| 两个分支计算量差异大 | `jcc` | `cmov` 要算两边，浪费 CPU |
| 可能触发异常的表达式 | `jcc` | `cmov` 两边都会执行 |

#### 缺陷对比

**条件跳转 `jcc` 的缺陷：**

- 现代 CPU 使用流水线，分支预测失败会清空流水线，造成性能损失
- 频繁的条件跳转可能影响 CPU 分支预测效率

**条件传送 `cmov` 的缺陷：**

- 两个分支的表达式都会计算，即使其中一个不会被使用
- 如果分支有副作用（如 `i++`、函数调用），不能安全使用
- 如果某个分支计算可能出错（如除以零、空指针解引用），也不能使用

> **一句话总结**：`jcc` 是"走哪条路"，`cmov` 是"两边都走完再选结果"。简单无副作用的条件赋值用 `cmov`，复杂或有副作用的条件用 `jcc`。
